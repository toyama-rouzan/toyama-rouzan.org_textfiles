　はじめに
　山岳会は入退会も頻繁にあり担当者が退会したあとでホームページが休止状態という事はよくあるケースです。このマニュアルはそんな事態にならないように、少しパソコンの知識があれば管理できるマニュアルの必要性を感じ作成したものです。
　とはいえ私自身パソコンの専門家でもなく、ましてやホームページ制作などを生業としてきたわけではありません。そんなことから自分の我流的な内容が含まれている可能性は大きく、また、富山労山のホームページの内容も徐々に変更されていくと思いますので、管理を引き継がれた方でお気づきの点があれば、ホームページ管理が継続可能なマニュアルに少しでも近づくよう改良されることを是非お願いします。

　2014年1月〜2月にかけてのホームページ改良は、特別なスキルが無くても通常の情報修正程度のホームページ変更は誰でも出来るような仕組みにすることで、ホームページの継続的管理を担保し、これによりホームページ担当者の負担を減らす事を主目的と考え作業を行いました。

具体的には
１、山行報告は表紙のリンクをその都度ホームページ担当者が作成していたが、山行報告ブログの一部を表示させることでブログ投稿が自動的に反映される様にした。
２、案内など対外的なお知らせを設置し、誰でも簡単に書き込みが出来るようにした。。
３、リンク集をCGIとし、誰でも簡単に変更出来る方式とした。
４、表紙写真はホームページ担当者が時折変更していたが、CGIを使ったスライドショー方式とし誰でも写真を追加、削除出来る方式とした。また、大量掲載によってオールラウンドである会の特徴や雰囲気が伝わりやすい表現方式となった。
５、連絡先として、メールアドレスを書いていたがフォームメールCGIによるものに変更した。問い合わせ種別の変更に関してはホームページ担当者のみ変更可能である。
６、会員専用ページを新設し会員の利便を図った。内容についてはCGIを多用することで通常はホームページ担当者が介在する必要はない。
７、借りているサーバースペースが１０GBと十分あるので、使用CGIは従前のレンタル掲示板などから自前サーバーに設置することとした。これによりＣＭ等も排除できたし、掲示板以外の多様な機能を実現出来た。
８、これらの改良により、ほとんどの変更は誰でも簡単にでき、このマニュアルに記載するような作業は大きな改良や修正が無い限り必要はないかもしれない。ホームページ担当者のありそうな作業は、あったとしてもKENTWE製ののCGIのパスワード変更と問い合わせ種別変更程度で済みそうで大きな省力化を果たした。

　会員専用メニューで必要と思われる項目についてはメニュー横に使い方などのリンクを作成している。各メニューの操作についてはどれも特に難しい事はないと思われるので、特にマニュアル化はしていない。

　以上の様な改良で今回大幅なノンスキル化、省力化が達成されたが、このマニュアルに書いたようなホームページ管理のスキルはやはり維持しておくべきで、興味のある方は是非取り組んでいただきたいと考えます。
なお、このマニュアルをホームページ作成、管理の学習材料とされる場合は　１、サーバーに接続　の前に　８、とりあえず無料サーバーなどで練習してみましょう　を参考にサーバーを借りて、１以降を見られるのも良い方法と思います。
　準備１　無料ホームページ作成ソフトを手に入れよう
　ホームページはHTML言語で書かれています。それではまずその勉強から・・・・いえいえ、知っているに越したことはありませんが、知らなくてもやりたいことをワープロの様に画面に作るだけで、HTML言語に変換してくれる便利なソフトがあります。

実際私もホームページビルダーという有料ソフトに頼っています。しかし、無料でも同じような機能を有するソフトがあります。まずこれをインストールしましょう。　　このマニュアルとあわせてご覧くださいフリーソフトでホームページを作ろう


HTMLエディター　alphaEDIT　

ここで大切なのはXPより後のwindowsをお使いの方はもう一つプログラムをインストールすることです。

　Windows 7/Vistaユーザーの方

　Windows 7/VistaではalphaEDITの動作にDhtmlEdというプログラムのインストールが必要です。これはMicrosoftが配布しているプログラムで公式ページからダウンロードできます。リンク先の公式ページは英語ページになっていますが、不安な場合は当サイトの下部リンクから直接ダウンロードできます。XP以前のユーザーの方はこのプログラムのインストールは不要です。

　ダウンロードしたDhtmlEd.msiを実行してインストールしていきます。英文の規約が表示され、同意する場合は「I Agree」にチェック。「I Agree」にチェックするとグレイアウトしていた「Next」ボタンがクリックできるようになるので「Next」でDhtmlEdのインストールが始まります。これでWindows 7/VistaでもalphaEditが使用できるようになります。

　６　CGIについて
　たとえばサーバーに写真やPDF化した会報をアップロードしようとすれば、いちいちFTPソフトを使いアップロードしなければならず、ホームページ管理者のみが作業可能となり、その負担は大きくなります。
しかし、CGIを使うことで誰でも簡単に目的を達成することが可能となり管理者の負担は軽減されます。
　サーバーによってはCGIの設置が出来ないものや、サーバー業者側で準備したCGIのみＯＫというものもあります。そうなるとホームページに備えたい機能が実現出来ないという事も起こりえますが、幸い富山労山が借りている　使えるネットの共用サーバー　はCGIを問題なく設置できます。CGIさえ入手できればよいわけです。

無料のＣＧＩはネット上でたくさんありますが、富山労山ＨＰ上では現時点（14年2月4日）でKENTWEBとcgi-designの物を利用しています。設置の方法や機能はそれらのHPで確認してください。

ホーム作成備忘録より
　会員専用コーナーCGI設置について使えるネットは初期設定では掲示板などのＣＧＩを設置しようとすると、専用のフォルダー（おそらくcgi-bin）でなければ動かない。これだとアクセス制限を掛ける会員専用には使えない。そこでネットで調べ下記の方法により他のフォルダーでも自由に設置できるようにした。

1)ホームページのルートフォルダーであるhttpdocsの.htaccess ファイルに以下の３行を書き込んだ。
Options +ExecCGI
AddHandler cgi-script .cgi 　　　　　　　この3行の記述は絶対に変えないでください。すべてのCGIが動かなくなります。
AddHandler cgi-script .pl

　※.htaccess  はサーバーの設定ファイルで不用意に書き換えると動作がおかしくなるので、書き換える時は必ずバックアップを取ったうえでお願いします。
.htpasswd はIDとパスワードの設定ファイルで定義されていればそのフォルダー以下は認証エリアになります。


2）作成された［.htaccess］のパーミッションが［644］であることを確認する。
3）上位ディレクトリにデフォルトで設置されている［cgi-bin］はそのまま残しておく。 　
ちなみに、CGIで使用するPerlのパスは、
#!/usr/bin/perl

sendmailのパスは、
/usr/sbin/sendmail

　※Perlパス、sendmailパスは使用サーバーに合わせる必要があります。上記が富山労山の使えるネットサーバーのパスです。



以上で専用フォルダーでなくてもCGIが動くようになった。

※　cgiのソースやテキストファイルなどの編集はメモ帳だとうまくいかないケースもあるとの指摘がネット上であるので出来ればエディターソフトを使った方が良いと思われる。

　init.cgi を TeraPadエディタ で開いて必要な箇所を修正します。この設定は14年2月4日現在の状態です。今後変更の可能性がありそうな箇所を赤色にしておきます。
 

# モジュール宣言/変数初期化
use strict;
my %cf;
#┌─────────────────────────────────
#│ JOYFUL NOTE : init.cgi - 2014/01/05
#│ Copyright (c) KentWeb
#│ http://www.kent-web.com/
#└─────────────────────────────────
$cf{version} = 'JoyfulNote v5.2';
#┌─────────────────────────────────
#│ [注意事項]
#│ 1. このスクリプトはフリーソフトです。このスクリプトを使用した
#│    いかなる損害に対して作者は一切の責任を負いません。
#│ 2. 設置に関する質問はサポート掲示板にお願いいたします。
#│    直接メールによる質問は一切お受けいたしておりません。
#└─────────────────────────────────
#===========================================================
# ■ 設定項目
#===========================================================
# 管理者用パスワード
$cf{password} = '0123';
# サムネイル画像を作成する（要：Image::Magick）
# → 縮小画像を自動生成し、画像記事の表示速度を軽くする機能
# 0=no 1=yes
$cf{thumbnail} = 0;
# 掲示板タイトル
$cf{bbs_title} = "情報交換掲示板";
# アップロードファイル管理者チェック機能 (0=no 1=yes)
# → アップロードファイルは管理者がチェック後表示
$cf{img_check} = 0;
# 最大記事数 (親記事+レス記事も含めた数）
$cf{max} = 50;
# 返信がつくと親記事をトップへ移動 (0=no 1=yes)
$cf{topsort} = 1;
# 画像の位置
# 1 : コメントの下
# 2 : コメントの左（廻り込み）
$cf{image_position} = 1;
# ミニカウンタの設置
# → 0=no 1=テキスト 2=GIF画像
$cf{counter} = 1;
# ミニカウンタの桁数
$cf{mini_fig} = 6;
# テキストのとき：ミニカウンタの色
$cf{cntcol} = "#bb0000";

　７、CGIのパーミッションについて
　パーミッションとは ?

 　パーミッション (permission) とは「許可」「認可」を意味する言葉です。
　サーバとして用いられるコンピュータは複数の人たちが同時に利用する性格のものであるため、お互いが他人に対して迷惑をかけたり、邪魔をしたりすることができないように、ファイルやホルダ（ディレクトリ）ごとに自分が読む権利、書いたり消したりする権利、実行する権利を個別に設定することができるようになっています。これらの権利の設定のことを指してパーミッションと呼びます。
　 パーミッションを適切に設定しておけば、ファイルを消したり書き換えられたりすることを防止することができます。 

　一つのファイル又はディレクトリに対して、オーナー（そのファイルの所有者）、グループ（同じグループのメンバー）、全員（その他の第三者）、という３種類のユーザー毎に、読む・書く・実行の３種類のパーミッションを設定することができます。
　 実行とはファイルの場合はプログラムやスクリプトなどを実行する権利、ホルダ（ディレクトリ）の場合はそのホルダ（ディレクトリ）の中に移動する権利のことを指します。

　　こう書くとややこしいですが、以下に図を書いて示してみます。

　　  　　　読む 書く 実行
　　所有者　 ○　 ○　 ○
　　グループ ○　 ○　 ○
　　全　員　 ○ 　○　 ○
　　　　　　○を付けた９つの部分に対して、それぞれ許可するかどうかを設定します。

　 所有者はもちろんあなた自身です。しかし、あなたが所有者であるとサーバが判断してくれるのは、FTPソフトウェアなどを使って、ID とパスワードを入力してログインしてきた場合だけです。NetscapeNavigator などのブラウザを使ってホームページにアクセスしてきた場合には、あなた自身も第三者と判断されます。
　 グループはサーバ管理者によって分けられていますが、第三者と同じ扱いと思って下さい。
　 従って、パーミッションを設定する際には、自分自身である所有者と、ホームページにアクセスしてくる人たちである第三者とを、ファイルやホルダ（ディレクトリ）毎に区別して設定しておけば、ファイルやホルダ（ディレクトリ）を守ることができます。

<2017/01/27補足>
　2017/01/12以降、CGIファイルおよびCGIファイルが入っているディレクトリのパーミッション設定において、「所有者」以外の書き込み権限が許可されていると、CGIを実行した時にエラー(500 Internal Server Error)が発生するようになりました。
　これは、サーバ管理会社においてWebサーバにセキュリティ強化対策としてsuExecを導入したためです。
　もし今後「CGIディレクトリを作り直したら、なぜかCGIが動かなくなった」といった現象が起きたときは、CGIファイルとそのディレクトリのパーミッションが「777」（全ユーザにすべて許可）などになっていないか確認して、もしそうなっていたらパーミッションを「755」（所有者は全て許可で、グループと全員は読みと実行のみ）に変更してください。

　ここまでざっと読んだ方は、なんだか大変そうだなと感じたかもしれません。
確かに、CGIの設置や変更などはパソコンでは今までやったことが無さそうな内容で、パスとかパーミッションとかPerlとか難しそうな用語が出てきます。
そうは言っても個人のホームページを作っている方も多い世の中です。こんな面倒があれば多くの人は尻込みするでしょう。
実際は、掲示板があったとしてもレンタル掲示板やサーバー業者が準備した物にリンクを張る程度が多く、これは一旦設置すれば後の作業はありません。
　記事などの追加はここで紹介したフリーや有料ソフトで、ワープロ感覚で編集しFTPソフト(または編集ソフトに備わっているFTP機能）を利用してアップロードしているだけだと思います。
　したがって、ソフトで編集するスキルとFTPでアップロードするスキルがあれば特に問題なくホームページの管理は出来ます。
あまり構えずやってみることで自然と分かってきて、次はCGIの自前設置などスキルアップの意欲が湧いてくると思います。

　９、複数のパソコンで管理する場合の注意点
　複数の方がホームページの修正作業などを行う場合。
　一人の管理者でも、自宅のパソコンと事務所のパソコンなど複数のパソコンで作業を行う場合。

　以上のような場合はサーバーに接続したら、まず作業しているファイルをダウンロードしてください。これをしないと、パソコンに保管されているかなり古いファイルと置き換わってしまう可能性があります。
